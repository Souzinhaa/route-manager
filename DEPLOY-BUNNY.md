# Deploy on Bunny VPS

Production-optimized deployment for Bunny VPS with Nginx reverse proxy, SSL/HTTPS, and resource constraints.

## Architecture

```
┌─────────────────────────────────────┐
│     Client (HTTPS port 443)         │
└────────────────┬────────────────────┘
                 │
        ┌────────▼─────────┐
        │  Nginx Reverse   │
        │  Proxy (SSL/TLS) │
        └────────┬─────────┘
                 │
        ┌────────▼──────────────┐
        │  FastAPI App + SPA    │
        │  (internal port 8000) │
        └────────┬──────────────┘
                 │
        ┌────────▼──────────┐
        │  PostgreSQL 15    │
        │  (internal)       │
        └───────────────────┘
```

## One-Command Deploy

```bash
curl -fsSL https://raw.githubusercontent.com/Souzinhaa/route-manager/deploy/bunny-vps/install-bunny.sh | bash
```

With GitHub token (optional):
```bash
export GH_USER="your_user" GH_TOKEN="your_token"
curl -fsSL https://raw.githubusercontent.com/Souzinhaa/route-manager/deploy/bunny-vps/install-bunny.sh | bash
```

## Manual Deploy (existing server)

```bash
cd /opt/route-manager  # or your target dir

# Clone
git clone -b deploy/bunny-vps https://github.com/Souzinhaa/route-manager.git .

# Setup
cp .env.example .env
nano .env  # Edit secrets + POSTGRES_PASSWORD

# Certificates
mkdir -p certs
openssl req -x509 -newkey rsa:2048 -keyout certs/key.pem -out certs/cert.pem \
  -days 365 -nodes -subj "/CN=yourdomain.com"

# Deploy
docker compose -f docker-compose.bunny.yml up -d
```

## Configuration

### .env

Key vars for Bunny:
```bash
POSTGRES_PASSWORD=strong_db_password_here
SECRET_KEY=strong_jwt_secret_32chars_minimum
GOOGLE_MAPS_API_KEY=your_api_key
APP_URL=https://yourdomain.com
```

### Nginx Config

File: `nginx-bunny.conf`

Features:
- HTTP → HTTPS redirect
- TLS 1.2 + 1.3
- Rate limiting (10 req/s for `/api`, 100 req/s general)
- Gzip compression
- Security headers (HSTS, CSP, X-Frame-Options)
- Upstream keepalive

Modify `server_name _;` to match your domain.

### Docker Compose

File: `docker-compose.bunny.yml`

Optimized for 2-4GB VPS:
- **DB**: 192M limit, 64M reservation
- **App**: 256M limit, 96M reservation
- **Nginx**: Shared cache volume
- Resource isolation via cgroups

## SSL/HTTPS Setup

### Let's Encrypt (Recommended)

```bash
# Install certbot
sudo apt-get install -y certbot python3-certbot-nginx

# Get certificate (requires port 80 open + DNS live)
sudo certbot certonly --standalone -d yourdomain.com

# Copy to app
sudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem certs/cert.pem
sudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem certs/key.pem
sudo chown $USER:$USER certs/*.pem

# Restart nginx
docker compose -f docker-compose.bunny.yml restart nginx
```

Auto-renewal:
```bash
sudo certbot renew --nginx --quiet --no-eff-email
```

### Self-Signed (Development)

Already generated by `install-bunny.sh`. Browser will warn — fine for testing.

To regenerate:
```bash
openssl req -x509 -newkey rsa:2048 -keyout certs/key.pem -out certs/cert.pem \
  -days 365 -nodes -subj "/CN=yourdomain.com"
docker compose -f docker-compose.bunny.yml restart nginx
```

## Firewall

Open on Bunny VPS / host firewall:
- **Port 80**: HTTP (redirect to HTTPS)
- **Port 443**: HTTPS (app)

Close:
- **Port 5432**: PostgreSQL (internal only)
- **Port 8000**: FastAPI (behind Nginx)

Example ufw:
```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

## Troubleshooting

### "Address already in use" (port 80/443)

Another service on VPS using ports:
```bash
sudo lsof -i :80
sudo lsof -i :443
```

Stop conflicting service or change Nginx listen ports in `docker-compose.bunny.yml`.

### Certificate errors in browser

Browser cached self-signed cert:
1. Refresh in incognito/private window
2. Or replace with Let's Encrypt cert

### App unhealthy

```bash
docker compose -f docker-compose.bunny.yml logs app --tail=50
```

Check DB connection:
```bash
docker compose -f docker-compose.bunny.yml exec app curl http://localhost:8000/health
```

### Database connection failed

Check env vars:
```bash
grep POSTGRES_PASSWORD .env
grep DATABASE_URL .env
```

Must match `docker-compose.bunny.yml` db service `POSTGRES_PASSWORD`.

## Resource Optimization

Bunny VPS typically has 2-4GB RAM. Current limits:
- DB: 192M
- App: 256M
- OS + Nginx: ~300M
- Free: ~1.2GB

To reduce further, edit `docker-compose.bunny.yml`:
```yaml
deploy:
  resources:
    limits:
      memory: 128M  # instead of 256M
```

Or increase if you have more RAM.

## Updating

```bash
cd route-manager
git fetch origin deploy/bunny-vps
git rebase origin/deploy/bunny-vps
docker compose -f docker-compose.bunny.yml build --no-cache
docker compose -f docker-compose.bunny.yml up -d
```

## Backups

### Database

```bash
# Dump PostgreSQL
docker compose -f docker-compose.bunny.yml exec db pg_dump -U postgres route_manager > backup.sql

# Restore
cat backup.sql | docker compose -f docker-compose.bunny.yml exec -T db psql -U postgres
```

### Uploads

```bash
# Copy volumes
docker run --rm -v route_manager_uploads_data:/uploads -v $(pwd):/backup \
  alpine tar czf /backup/uploads.tar.gz -C /uploads .

# Restore
docker run --rm -v route_manager_uploads_data:/uploads -v $(pwd):/backup \
  alpine tar xzf /backup/uploads.tar.gz -C /uploads
```
